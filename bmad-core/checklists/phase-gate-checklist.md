# Phase Gate Checklists

## Discovery -> Analysis Gate
- [ ] Project brief completed
- [ ] Vision statement defined
- [ ] Stakeholders identified
- [ ] High-level scope agreed
- [ ] Success criteria defined
- [ ] Initial risks documented

## Analysis -> Architecture Gate
- [ ] Requirements document complete
- [ ] User stories written with acceptance criteria
- [ ] Requirements prioritized (MoSCoW)
- [ ] Domain model created
- [ ] Open questions resolved or tracked
- [ ] Stakeholder sign-off on requirements

## Architecture -> Planning Gate
- [ ] System design document complete
- [ ] Key ADRs documented
- [ ] Technology stack selected
- [ ] Data model designed
- [ ] API contracts defined
- [ ] Security considerations documented
- [ ] Architecture reviewed and approved

## Planning -> Implementation Gate
- [ ] Sprint backlog created
- [ ] Tasks estimated
- [ ] Dependencies mapped
- [ ] Team assignments made
- [ ] Definition of done agreed
- [ ] Development environment ready

## Implementation -> Testing Gate
- [ ] All sprint tasks complete
- [ ] Code reviewed
- [ ] Unit tests passing
- [ ] Code merged to main branch
- [ ] Build succeeding

## Testing -> Review Gate
- [ ] Test plan executed
- [ ] All critical/high bugs fixed
- [ ] Test coverage meets target
- [ ] Performance benchmarks met
- [ ] Security scan clean

## Review -> Deployment Gate
- [ ] Sprint review completed
- [ ] Retrospective completed
- [ ] Release notes drafted
- [ ] Deployment checklist prepared
- [ ] Rollback plan documented
- [ ] Stakeholder approval received
